# i18n Design Spec

## Goal

Add internationalization to AGI Rush so the game can be played in 8 languages: English (default), French, Italian, German, Spanish, Polish, Simplified Chinese, and Russian.

## Libraries

- `i18next` — core i18n framework
- `react-i18next` — React bindings (`useTranslation` hook, `<Trans>` component)
- `i18next-browser-languagedetector` — auto-detect browser language on first visit

All added to `apps/game/package.json` only.

## Translation File Structure

```
apps/game/src/i18n/
├── index.ts                    # i18next init + config
└── locales/
    ├── en/
    │   ├── ui.json             # UI labels, buttons, status text
    │   ├── upgrades.json       # Upgrade names & descriptions
    │   ├── tech-tree.json      # Tech node names & descriptions
    │   ├── tiers.json          # Tier names & taglines
    │   ├── events.json         # Event names, descriptions, choice labels
    │   ├── milestones.json     # Milestone names & descriptions
    │   ├── ai-models.json      # Special ability text, agent setup text
    │   └── tutorial.json       # Tutorial screen text
    ├── fr/
    │   └── (same files)
    ├── it/
    ├── de/
    ├── es/
    ├── pl/
    ├── zh/
    └── ru/
```

## Namespaces

| Namespace | Source | Key pattern | Example |
|-----------|--------|-------------|---------|
| `ui` | Hardcoded JSX strings | `section.label` | `ui:resource_bar.queued` → "Queued" |
| `upgrades` | `upgrades.json` entity ids | `{id}.name`, `{id}.description` | `upgrades:freelancer.name` → "Freelancer" |
| `tech-tree` | `tech-tree.json` entity ids | `{id}.name`, `{id}.description` | `tech-tree:auto_type.name` → "Auto-Type" |
| `tiers` | `tiers.json` tier ids | `{id}.name`, `{id}.tagline` | `tiers:garage.tagline` → "A keyboard and a dream." |
| `events` | `events.json` entity ids | `{id}.name`, `{id}.description`, `{id}.options.{label}` | `events:production_down.name` |
| `milestones` | `milestones.json` entity ids | `{id}.name`, `{id}.description` | `milestones:hello_world.name` |
| `ai-models` | `ai-models.json` entity ids | `{id}.special.name`, `{id}.special.description`, `{id}.agent_setups.{index}` | `ai-models:claude_haiku.special.name` |
| `tutorial` | Tutorial screen | `step.{key}` | `tutorial:tip_1` |

## i18n Init (`apps/game/src/i18n/index.ts`)

```typescript
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import LanguageDetector from "i18next-browser-languagedetector";

// English bundles imported statically (default, always available)
import enUi from "./locales/en/ui.json";
import enUpgrades from "./locales/en/upgrades.json";
// ... all en namespaces

// Non-English locales loaded lazily
const lazyLocales: Record<string, () => Promise<Record<string, unknown>>> = {
  fr: () => import("./locales/fr"),
  it: () => import("./locales/it"),
  de: () => import("./locales/de"),
  es: () => import("./locales/es"),
  pl: () => import("./locales/pl"),
  zh: () => import("./locales/zh"),
  ru: () => import("./locales/ru"),
};

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: "en",
    defaultNS: "ui",
    ns: ["ui", "upgrades", "tech-tree", "tiers", "events", "milestones", "ai-models", "tutorial"],
    resources: {
      en: {
        ui: enUi,
        upgrades: enUpgrades,
        // ... all en namespaces
      },
    },
    interpolation: { escapeValue: false }, // React already escapes
    detection: { order: ["localStorage", "navigator"], caches: ["localStorage"] },
  });
```

## Component Integration

### Hook usage

```typescript
// In a component
const { t } = useTranslation("ui");
return <div>{t("resource_bar.queued")}</div>;

// For data entities
const { t } = useTranslation("upgrades");
const name = t(`${upgrade.id}.name`);
const description = t(`${upgrade.id}.description`);
```

### Interpolation

```json
{ "loc_rate": "{{rate}} loc/s" }
```

```typescript
t("resource_bar.loc_rate", { rate: formatNumber(locRate) })
```

### Data entity rendering

Components that render domain entities (upgrades, tech nodes, events, etc.) will call `t()` with the entity's `id` as the key prefix instead of reading `entity.name` / `entity.description` directly. The domain JSON files keep their English text as documentation, but the rendered text comes from translation files.

## Language Selector

Added to the existing settings page (both desktop and mobile):

- Dropdown/select with language names in their native script: English, Français, Italiano, Deutsch, Español, Polski, 中文, Русский
- Changing language calls `i18n.changeLanguage(code)` which persists to localStorage via the detector
- No page reload needed — react-i18next re-renders reactively

## What Stays in English

- **Singularity sequence** — narrative monologue (~900 lines), deferred to a future pass
- **AI model names** — Claude, GPT, Gemini, etc. are real product names
- **Code editor tokens** — flavor text in the typing area
- **CLI prompt flavor text** — cosmetic terminal output

## Fallback Behavior

- Missing key in current language → falls back to English
- Missing namespace → falls back to English
- This means partial translations are safe — untranslated strings show in English, not as raw keys

## No-Scope Items

- No pluralization rules (number formatting handles this)
- No RTL support (no RTL languages in the set)
- No server-side rendering concerns (SPA only)
- No translation management platform integration
- Translations are AI-generated initial drafts, refinable later
